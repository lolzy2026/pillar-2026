## 2. Core Components Detail

### 2.1 Component Responsibility Matrix

| Component | Responsibility | Technology | Data Stored |
|-----------|---------------|------------|-------------|
| **Chat UI** | User interaction, form display, message rendering | React | None (stateless) |
| **FastAPI Endpoints** | HTTP routing, request validation, response formatting | FastAPI | None |
| **LangGraph Agent** | Multi-step reasoning, node orchestration, checkpointing | LangGraph | Graph state in checkpoint |
| **MCP Tool Node** | Tool execution, SSE streaming, elicitation detection | Python async | None (uses Redis) |
| **Elicitation Session Manager** | Lifecycle management, timeout handling, callback coordination | Python async | Elicitation metadata in Redis |
| **Redis** | Hot data storage, TTL management, pub/sub | Redis 7+ | Sessions, elicitations, executions |
| **PostgreSQL** | Permanent storage, audit trail, conversation history | PostgreSQL 14+ | Messages, tool logs, elicitation events |
| **MCP Gateway** | Tool routing, auth/authz, namespace filtering | External Service | Tool schemas, execution state |

## 12. Critical Design Decisions Summary

| Decision Point | Choice | Rationale |
|---------------|--------|-----------|
| **Elicitation Node** | Same node as MCP execution | Atomic operation, maintains context, cleaner graph |
| **Session Storage** | Redis for hot data, PostgreSQL for cold | Performance + persistence balance |
| **Response Endpoint** | Dedicated `/elicitation/respond` | Clear separation, better error handling |
| **Timeout Duration** | 3 minutes | Balance UX vs resource usage |
| **Parallel Elicitations** | Display all simultaneously | Better UX, faster completion |
| **Graph Pause Mechanism** | LangGraph checkpoint + exception | Leverages framework, resilient |
| **Event Buffering** | Redis with 50-event limit | Enables reconnection, minimal storage |
| **SSE Connection** | Per tool execution | Simpler lifecycle, easier debugging |

## 13. Risk Mitigation

| Risk | Mitigation Strategy |
|------|-------------------|
| **User abandons form** | Timeout handler cleans up after 3 min |
| **Backend crashes during elicitation** | Redis + checkpoint persistence enables resume |
| **MCP Gateway unavailable** | Retry logic, user-friendly error messages |
| **Redis failure** | Redis Cluster with failover, graceful degradation |
| **Multiple elicitations overwhelm user** | Sequential or parallel display with clear queue |
| **Long-running tool executions** | SSE streaming keeps connection alive, event buffering |
